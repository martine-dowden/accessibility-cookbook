Links and Buttons: neither are divs or spans

Overview

Link: An interactive reference to an internal or external resource that, when activated, causes the user agent to navigate to that resource.

Button: An input that allows for user-triggered actions when clicked or pressed.

Accessible Rich Internet Applications (WAI-ARIA) 1.0: The Role Model

What am I?

Click me to find out.

Click Me

I'm actually a div, and if you try to tab to me you will realize that I cannot be tabbed too via keyboard.

The problem with using <div> or <span> when a link or a div would be appropriate is the lack of availability in taborder, making them lack keyboard accessibility without extra work. Furthermore, <div> and <span> do not expose the approapriate attributes to the scren readers.

Requirement <div> & <span> <a> & <button>
Natively keyboard accessible No Yes
(for <a>, only when href present)
Sufficient Color Contrast Design Dependent Design Dependent
(unless using native BUTTON with no CSS)
Exposes appropriate name, state, role, value No Yes

How to choose?

Since you are navigating, use a Link.
