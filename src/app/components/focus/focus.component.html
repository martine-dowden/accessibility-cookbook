Focus

Overview & Demo

Focus Visible: Any keyboard operable user interface has a mode of operation where the keyboard focus indicator is visible. (Level AA)

WCAG 2.0 Success Criteria 2.4.7

Having a visual indicator of the screen element currently in focus helps users who rely on keyboard to operate the page know with which operation they will interact at any point in time.

Demo

Click in the box and then tab through these elements. Notice the difference in experience when element in focus is indicated versus when it is not.

HTML & CSS

"G195: Using an author-supplied, highly visible focus indicator"

spec-techniques for WCAG 2.0: G195

Example

Code

<button type="button" class="demo-button">Demo Button</button>
.demo-button {
  padding: .75rem 1rem;
  background: $primary;
  color: white;
  font-size: 1rem;
  border: solid .0625rem $primary;
  border-radius: .125rem;
  cursor: pointer;
  box-shadow: 0 .0625rem .1875rem 0 rgba(0,0,0,0.12),
              0 .0625rem .125rem 0 rgba(0,0,0,0.24);
}

.demo-button:hover,
.demo-button:focus {
  background: lighten($primary-light, 35%);
  color: $primary;
  border: solid .0625rem lighten($primary-light, 35%);
}
